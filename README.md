# NKRlive

A real-time civic dashboard for Visakhapatnam (Vizag) — local time, weather,
a dedicated Nitish Kumar Reddy tracker, and cross-verified local news, pushed
to the browser instantly over WebSockets.

## Features

- **Vizag time** for Asia/Kolkata, updated every second.
- **Weather** via the [Open-Meteo](https://open-meteo.com/) API
  (no API key required).
- **Live news feed** aggregated from Google News, Times of India, The Hindu
  (Andhra Pradesh), and optionally Twitter/X, polled every ~12 seconds and
  pushed to connected clients as soon as new items appear.
- **Cross-source verification** — a story is flagged "verified" once it has
  appeared via two or more independent sources.
- Category filters and headline search on the front end.

## Getting started

```bash
npm install
npm start
```

The server listens on `PORT` (default `3000`) and binds to `HOST` (default
`0.0.0.0`). Open `http://localhost:3000` in a browser.

## Configuration

| Variable               | Default     | Description                                              |
|-------------------------|-------------|-----------------------------------------------------------|
| `PORT`                  | `3000`      | HTTP/WebSocket port                                        |
| `HOST`                  | `0.0.0.0`   | Bind address                                               |
| `TWITTER_BEARER_TOKEN`  | _(unset)_   | Optional. Enables Twitter/X search (see `config.js`)      |
| `ANTHROPIC_API_KEY`     | _(unset)_   | Optional. Enables the NKR AI live situation report (see `config.js`) |

## API endpoints

| Endpoint        | Description                                              |
|------------------|-----------------------------------------------------------|
| `GET /api/news`    | Most recent cached news items (JSON)                     |
| `GET /api/weather` | Current weather & forecast passthrough (Open-Meteo)      |
| `GET /api/aqi`     | Current air quality passthrough (Open-Meteo)              |
| `GET /api/health`  | Liveness check + process uptime                          |
| `GET /api/status`  | Poll diagnostics: last poll time, errors, cached items, connected clients |
| `GET /api/nkr-status` | NKR's detected current location, live weather there, and next 5 upcoming matches. Add `?refresh=1` to force re-check. |
| `GET /api/nkr-ai`  | Live NKR situation report generated by Claude (web search). Add `?refresh=1` to force regeneration. Returns `{text, error, generatedAt}` |
| `WS /`             | WebSocket stream — `{type:"init", items:[...]}` on connect, then `{type:"news", item:{...}}` for each new story |

## Notes

- The in-memory dedupe cache and news cache are capped (`MAX_SEEN_HASHES`,
  `MAX_ITEMS_STORED` in `server.js`) so long-running processes don't grow
  unbounded.
- The server pings WebSocket clients every 30 seconds and drops connections
  that stop responding.

## NKR Live Status (location, weather, next 5 matches)

The NKR Spotlight panel shows three things, all derived from data already
being collected — no AI / API key required:

1. **Current location** — the server scans recent `nkr` and `sports`
   category headlines for known venue/city names (see `VENUES` in
   `server.js`) and picks the most recent match. If nothing matches, it
   defaults to Visakhapatnam (home base).
2. **Live weather there** — once a location is detected, its coordinates
   are used to fetch current conditions from Open-Meteo (temperature,
   feels-like, humidity, wind).
3. **Next 5 matches** — a maintained fixture list (`NKR_FIXTURES` in
   `server.js`) covering the current India vs Afghanistan ODI series and
   the upcoming England tour. Past matches are automatically filtered out.

Served via `GET /api/nkr-status`, cached for 5 minutes (force-refresh with
`?refresh=1`). The front end refreshes automatically every 5 minutes.

**Keeping fixtures current**: edit the `NKR_FIXTURES` array in `server.js`
when new series are announced or dates change — each entry needs a date,
time, opponent/match label, format, and a `venueKey` matching an entry in
`VENUES` (add new venues there as needed).

| Reddit r/Cricket & r/IndianCricket | Fan-detected sightings, match-day posts |

### Confidence scoring system
Location detection now produces a **confidence score (0–99%)** combining:
- **Source reliability** — BCCI/ESPNcricinfo/presser = 85–92%; Instagram/YouTube = 75%; Reddit = 54%  
- **Recency decay** — <2 hours: +10 pts; >48 hours: −28 pts
- **Multi-source bonus** — up to +12 pts when 3+ independent sources agree

The NKR page now shows:
- A colour-coded confidence bar (green ≥70%, amber ≥40%, red <40%)
- Source count ("4 signals detected")
- An expandable breakdown table with per-signal city, source name, confidence %, and age

### No new API keys required
Instagram scraping uses `picuki.com` (public viewer). Twitter/X remains optional (set `TWITTER_BEARER_TOKEN` in environment).

## v1.6 — API-Powered Intelligence (NewsAPI · CricAPI · YouTube · Gemini)

### New API integrations

| API | Key env var | Free tier | What it adds |
|-----|-------------|-----------|--------------|
| **NewsAPI** | `NEWS_API_KEY` | 100 req/day | Latest English articles mentioning NKR — title + source for location extraction |
| **CricAPI** | `CRIC_API_KEY` | 100 req/day | Live match data — if NKR is in a playing XI today, the venue is a near-certain location fix |
| **YouTube Data API v3** | `YOUTUBE_API_KEY` | 10,000 units/day | Recent uploads — video titles & descriptions contain stadium/city name clues |
| **Gemini 1.5 Flash** | `GEMINI_API_KEY` | Free tier at aistudio.google.com | Synthesises all evidence (headlines + CricAPI + fixtures) into a structured verdict with city, confidence %, and 1–2 sentence reasoning |

### How Gemini inference works
Every 10 minutes (configurable via `GEMINI_CACHE_MS`) the server sends Gemini a prompt containing:
- Up to 20 recent NKR-tagged headlines with source names
- Any live CricAPI match venue string
- The next 3 fixtures from `NKR_FIXTURES`

Gemini responds with JSON: `{ city, country, venue, confidence, reasoning, primarySignal }`.

The final confidence shown on the page is `Math.max(rule-based score, Gemini score)` — whichever is higher wins.

### New UI elements in nkr.html
- **Gemini AI card** (blue tinted) — shows Gemini's city verdict, confidence %, reasoning sentence, and the single strongest evidence piece it cited
- **CricAPI live badge** (red pulsing dot) — appears when a live match confirms the venue via CricAPI

### Quick start
```bash
# Set keys before starting the server:
export NEWS_API_KEY=your_key_here
export CRIC_API_KEY=your_key_here
export YOUTUBE_API_KEY=your_key_here
export GEMINI_API_KEY=your_key_here
npm start
```
All keys are optional — the app degrades gracefully to RSS/rule-based scoring if any key is missing.

### New capabilities

#### Google Custom Search API (`GOOGLE_CSE_API_KEY` + `GOOGLE_CSE_CX`)

| Query | Target |
|-------|--------|
| `"Nitish Kumar Reddy" site:instagram.com` | Instagram public posts |
| `"Nitish Reddy" site:instagram.com` | Instagram short name |
| `#NitishKumarReddy site:x.com` | X/Twitter hashtag |
| `"NKR" cricket site:x.com` | X NKR abbreviation |
| `"Nitish Kumar Reddy" site:youtube.com` | YouTube videos |
| `"Nitish Reddy" site:facebook.com` | Facebook public pages |

Get a free Programmable Search Engine at **cse.google.com** (100 free queries/day).

#### Facebook Public Page RSS (`fetchFacebookRSS`)
Polls RSS proxy feeds for SRH, BCCI, and Cricbuzz Facebook pages.
Items are filtered to only those mentioning "Nitish" — no API key needed.

#### Gemini Vision — image landmark detection (`analyseImageWithGemini`)
For each new NKR news article, the server fetches the page's `og:image`,
sends it to Gemini 1.5 Flash Vision, and asks it to identify:
- Cricket stadiums and sports venues
- City landmarks, mountains, distinctive architecture
- Hotel logos and team hotel clues
- On-screen text showing a location

Results with confidence ≥ 40% become a `"Gemini Vision (image analysis)"` signal
in the location engine — shown with a 🔭 Vision chip in the Evidence Flow panel.

#### Fan Signal Boost (`computeFanSignalBoost`)
Each candidate city now gets a **diversity bonus** (up to +20 pts) based on
how many *different tiers* of sources agree within the last 6 hours:

| Condition | Bonus |
|-----------|-------|
| ≥1 official signal (BCCI/CricAPI/SRH/presser) | +8 |
| ≥1 media signal (ESPN/Cricbuzz/NewsAPI/TOI) | +5 |
| ≥2 fan signals (Instagram/X/Reddit/YouTube/CSE) | +4 |
| All three tiers present | +5 extra |

This mirrors the fan-post logic described: Fan photo + Team post + News article → confident location.

## v2.1 — NKR Match Day Tracker

A dedicated match-day section on the NKR page (between "Where is NKR Right Now" and "Upcoming Fixtures") that activates automatically on confirmed India match days. All data comes from CricAPI — the section is completely hidden on non-match days.

### Five real-time phases

| Phase | Trigger | What you see |
|---|---|---|
| **Pre-toss** | Match day, toss not yet done | 🟡 Awaiting Toss + live countdown to scheduled start |
| **XI Pending** | Toss done, Playing XI not yet confirmed | 🔎 Confirming Playing XI… |
| **Not Playing** | NKR confirmed absent from XI | 🔴 NKR NOT PLAYING TODAY + official reason (if set by admin) |
| **Playing Live** | NKR in XI, match in progress | 🟢 Live batting / bowling / catches auto-refreshing every 15s |
| **Completed** | Match over, NKR played | 📊 Full performance card — visible for 12 hours then auto-hidden |

### Real-time notifications (toast popups via vlShowToast)
- NKR confirmed in / not in Playing XI
- NKR comes to bat
- Batting milestones: 25, 50, 100 runs
- Bowling wickets: 1, 2, 3, 4-for, 5-for
- Catch taken

### Never-guess rule
NKR's non-selection reason is **never auto-generated**. Once you have a verified official reason (BCCI announcement, press conference, broadcaster confirmation), set it via the protected admin endpoint:

```bash
curl -X POST http://localhost:3000/admin/matchday/reason \
  -H "Content-Type: application/json" \
  -H "x-admin-pass: $ADMIN_PASS" \
  -d '{"reason":"Rested"}'

# View the current reason + reference categories
curl -H "x-admin-pass: $ADMIN_PASS" http://localhost:3000/admin/matchday/reason
```

Reference categories: Injury · Rested · Team Combination · Tactical Decision · Workload Management

### Requirements
Set `CRIC_API_KEY` (cricapi.com, free tier 100 req/day) for live toss, Playing XI, batting, bowling and fielding data. Without the key the section still shows the match name and countdown — it just can't poll live data.
